# GitHub → Vercel 部署流程

## 事前確認
- GitHub 已登入：`gh auth status` → `ericsopenclaw`
- Vercel 已登入：`vercel whoami` → `ericsopenclaw-3554`

---

## Step 1：初始化 Git Repo 並推上 GitHub

```bash
cd /Users/eric/.openclaw/workspace

# 在 GitHub 建立新 repo（假設專案名為 PROJECT_NAME）
gh repo create PROJECT_NAME --private --source=. --push

# 或指定目錄：
# gh repo create PROJECT_NAME --private --source=/path/to/project --push
```

---

## Step 2：Vercel 連結 Repo

**方法A（推薦）：用 Vercel 網站手動連結**
1. 前往 https://vercel.com/new
2. Import Git Repository → 選 `ericsopenclaw/PROJECT_NAME`
3. Framework Preset 選 `Other`（純 HTML/JS 專案）
4. Build Command / Output Directory 留空或填 `.`
5. Deploy → 拿到 `https://PROJECT_NAME.vercel.app` 網址

**方法B：用 Vercel CLI 連結**
```bash
cd /path/to/project
vercel link
vercel --prod
```

---

## Step 3：設定 Custom Domain（可選）

在 Vercel Dashboard → Settings → Domains：
- 可用 `shasha.maxc.tw` 的 subdomain
- 或用 Cloudflare Tunnel（`openclaw.eric020730.com` 已有的設定）

## 常用指令

```bash
# 看目前连结状态
vercel link

# 重新部署（觸發新一次 --prod）
vercel --prod

# 看部署網址
vercel inspect
```
